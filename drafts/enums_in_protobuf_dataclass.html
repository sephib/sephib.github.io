Intro

working with data many times requires generating a collection of data as some sort of object with internal relationship. A useful tool for managing such objects are dataclass Protobuf is a protocol (introduced by Google) for transferring data between different entities, in a reliable and efficient manner.

In this blog post I want to review the relationship between protobuf and dataclass.

Dataclasses Background

The Dataclasses are able to be used in two contexts: 1. working with data in a usefull maner 2. Working with Classes that can generate allot of the boiler plate that otherwise would require to work on.

In an essence the dataclass is a combination of a NamedTuple and a Class. The convenience of retriving data like NamedTuple while supercharging it with Class capabilities.
